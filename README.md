# Семинар 6. Музей

Проект по предметной области «Музей» для варианта 23.

В проекте сохранена функциональность предыдущих семинаров:
классы музея, залов, экспонатов, гида, посетителя, каталога и собственный
контейнер `MyContainer<T>`. Для семинара 6 добавлена очередь отложенных
вызовов функций и методов классов предметной области.

## Предметная область

В программе используются основные сущности музея:

- `Museum` — музей, хранит залы и коллекцию экспонатов;
- `Hall` — зал музея;
- `MuseumItem` — абстрактный базовый класс музейного объекта;
- `Exhibit` — общий базовый класс экспоната;
- `Painting` — картина;
- `Sculpture` — скульптура;
- `Artifact` — артефакт;
- `Guide` — гид с маршрутом экскурсии;
- `Visitor` — посетитель, который может работать с гидом;
- `Catalog` — каталог для поиска экспонатов;
- `MyContainer<T>` — собственный шаблонный контейнер.

## Что было сохранено из семинара 5

В проекте по-прежнему используется собственный контейнер `MyContainer<T>`,
а не `std::vector`.

`MyContainer<T>` поддерживает:

- динамическое хранение элементов;
- `push_back`;
- `pop_back`;
- доступ по индексу через `operator[]`;
- `size`;
- `empty`;
- копирование и перемещение;
- итераторы `begin()` и `end()`.

Контейнер применяется в предметной области:

- `Museum` хранит залы в `MyContainer<Hall>`;
- `Museum` хранит экспонаты в `MyContainer<std::shared_ptr<MuseumItem>>`;
- `Guide` хранит маршрут в `MyContainer<MuseumItem*>`;
- `Catalog` выполняет поиск по коллекции музея;
- прикладная задача строит короткий маршрут по залу.

Также в программе демонстрируются:

- полиморфизм через базовый класс `MuseumItem`;
- `std::find` для поиска зала;
- `std::sort` для сортировки маршрута;
- `typeid` и `dynamic_cast`;
- композиция, агрегация, ассоциация и делегирование.

## Семинар 6

Для семинара 6 добавлен файл `include/PendingQueue.h`.

В нём реализована очередь отложенных вызовов:

- `IPending` — базовый интерфейс с виртуальным методом `run()`;
- `Pending<Callable, Args...>` — шаблонный класс, который хранит вызываемый
  объект и аргументы;
- `PendingQueue` — очередь задач, которые можно выполнить позже.

Очередь использует:

- вариадические шаблоны `Args...`;
- `std::decay_t` для хранения callable и аргументов без проблем с временными
  объектами;
- `std::tuple` для хранения произвольного набора аргументов;
- `std::apply` для выполнения сохранённого вызова;
- `std::queue<std::unique_ptr<IPending>>` для хранения разных типов вызовов
  через общий интерфейс.

## Методы `PendingQueue`

```cpp
template<class Callable, class... Args>
void enqueue(Callable&& callable, Args&&... args);

void run();
void runAll();
bool empty() const;
std::size_t size() const;
```

Назначение методов:

- `enqueue` добавляет отложенный вызов функции или метода и его аргументы;
- `run` выполняет один вызов из очереди;
- `runAll` выполняет все оставшиеся вызовы;
- `empty` проверяет, пуста ли очередь;
- `size` возвращает количество отложенных вызовов.
